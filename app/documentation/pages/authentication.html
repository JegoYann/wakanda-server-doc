Authentication

Overview

  • Authentication inside Wakanda Server is handled by the directory (cf Wakanda Server Overview)
  • The directory offers a default authentication mechanism but gives the possibility to completely override it
  • A default REST API gives access to authentication and session management
  • An SSJS API is available for different kinds of authentication/session operations
  • Multiple authentication schemes can coexist on the same application

REST API

By default, Wakanda Server exposes a REST API so you can handle common directory operations. You can read more about the subject by visiting the doc-center (migration in progress).

SSJS API

An SSJS API is available through the globally exposed object directory to handle all kind of authentication and session management operations. The Directory interface documentation gives more details about the available methods and properties of this object.
