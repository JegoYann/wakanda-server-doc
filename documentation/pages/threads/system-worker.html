System Worker

Why

  • You need to execute an external application.
  • You need to retrieve the result of an external application.
  • You need to do it in a blocking or non blocking way.

What is it

  • A system worker is running in its own thread.
  • A system worker runs external system application.
  • A system worker runs synchronously or asynchronously.
