Custom Session Management

This is an enterprise version feature only.

By default, Wakanda handles user sessions and ensures :

  • the creation and in-memory persistence of session information
  • the auto-removal of expired sessions
  • the handling of SSJS session APIs
In your app, you might need to have full control over those aspects. Here are some use case examples :
  • You want to use multiple Wakanda Server instances behind a load balancer and need to share sessions across the servers
  • You want to use a different way than cookies to keep users authenticated (JWT for instance)
  • You want to log user login/logout activity

How to

In order to override the default session handling, you need to call the directory.setSessionManager function and pass in the name of your Wakanda SSJS module that will handle all session operations.

You would usually call directory.setSessionManager during the initialization of your project using a bootstrap or a service.
