Wakanda Server Overview

Wakanda Server is a multi-threaded server. The most important components of the server are :

  • JavaScript Application Server
    • V8 as JavaScript runtime
    • Rich API to handle filesystem, sockets, HTTP Requests, internal or remote databases, workers..
    • Since Wakanda SSJS is different than NodeJS' SSJS, you can use the NodeWorker API to start and use a NodeJS thread inside Wakanda Server
    • Debugger based on Chrome DevTools to debug the different running JavaScript contexts (one context per thread)
  • Wakanda HTTP Server
    • Serves static HTTP files for your applications
    • Exposes internal or remote databases through a default secure REST API
    • Can be used by your application's SSJS code to add custom HTTP patterns..
  • Data Abstraction Layer
  • WakandaDB (Internal DataBase)
  • Directory
